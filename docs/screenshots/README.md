# Screenshots

Place six PNGs here that the root README references:

| Filename | What it should show |
|---|---|
| `01-welcome.png` | The welcome / landing page after sign-in (or guest mode), showing the three practice-mode cards |
| `02-setup.png` | The 4-step setup wizard (info → category → difficulty → summary), ideally on the category or summary step |
| `03-session.png` | The 3-panel live interview workspace mid-session: question on the left, transcript / input in the middle, timer + tips on the right |
| `04-results.png` | The results screen with score ring + 5-dimension bars |
| `05-breakdown.png` | The per-question feedback (LLM or rule-based) expanded for a single question |
| `06-history.png` | The history view with the progress chart + recent sessions |

**How to capture:**

1. Run the app locally (see root README "Setup / Running Locally").
2. Resize the browser to roughly 1440×900 for a clean composition.
3. Use the OS screenshot tool (`Cmd+Shift+4` on macOS, snip area).
4. Save as the filenames above, PNG, into this folder.
5. Commit and push — they'll render automatically in the root README on GitHub.

You can capture in **either dark or light theme**, but pick one and stay consistent across all six.
